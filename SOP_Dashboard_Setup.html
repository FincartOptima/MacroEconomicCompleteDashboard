1. What This Dashboard Is For

Purpose

The India Economic Pulse dashboard is an internal macroeconomic monitoring tool built for the Fincart research team. It consolidates 35 key economic indicator series across 8 categories into a single, interactive view — replacing the need to visit dozens of government websites, RBI bulletins, and AMFI reports each month.

It serves three primary functions:

  1. Monthly macro tracking: At-a-glance KPI cards show the latest values for the six most critical indicators (Repo Rate, Credit Growth, CPI, GST, USD/INR, PMI), so you know the economic pulse in under 10 seconds.
  2. Trend analysis: Interactive line/bar charts let you spot multi-month trends, inflection points, and divergences across indicators.
  3. Data repository: Detailed tables with 21+ months of history, source URLs, and data tiers provide an auditable single source of truth for investment committee discussions, client presentations, and research reports.

Who Should Use It

  • Investment team: For monthly macro reviews, portfolio positioning, and sector allocation decisions
  • Research analysts: For sourcing data points for reports, screening macro themes, and back-testing narratives
  • Client-facing advisors: For quickly referencing current macro data during client meetings
  • Leadership: For a high-level economic snapshot without wading through raw data

2. Dashboard Architecture

The dashboard is a static website — two files, no server, no database.

ComponentRole
index.htmlThe complete dashboard — HTML, CSS, and JavaScript in one file. Contains all chart rendering, table layout, admin mode, refresh logic, and download functionality. Uses Tailwind CSS and Chart.js via CDN.
data.jsonThe single source of truth for all indicator data. Contains 34 raw indicators (35 display rows, since fuel splits into petrol and diesel) across 21+ months, with values, source URLs, data tiers, and notes.

How it works: When a visitor opens the page, index.html fetches data.json via the browser's fetch() API, processes it client-side using embedded indicator metadata, and renders KPI cards, trend charts, and data tables. No Python, no build step, no server.

Data flow:

data.json (on disk / GitHub)
    ↓  fetch()
rawData (in-memory JS object)
    ↓  processRawData()
INDICATORS / CATEGORIES / MONTHS (display arrays)
    ↓  renderAll()
KPI Cards + Charts + Tables (DOM)

3. How to Use the Dashboard

Viewing Mode (Default)

Anyone with the URL can view the dashboard. No login required for read-only access.

KPI Cards (Top Section)

Six summary cards show the latest available value for the most watched indicators: RBI Repo Rate, Bank Credit Growth, CPI Inflation, GST Collection, USD/INR, and Manufacturing PMI. Each card shows the value, its unit, and the month it corresponds to. Click any value to open the source URL in a new tab. Click the indicator name itself to open the detail drawer (see below).

Trend Chart (Middle Section)

A dropdown selector lets you plot any of the 35 indicator series as a time-series chart. Line charts are used for rates and percentages; bar charts for absolute volumes (flows, collections, consumption). Hover over data points to see exact values.

Data Tables (Bottom Section)

Eight category tabs display detailed monthly data in scrollable tables. Each row is one indicator, each column is one month (Oct 2024 onward). Click any cell value to open its source URL. Tables can be exported to Excel using the Export to Excel button on each category header.

Indicator Detail Drawer

Click any indicator name (in a KPI card or the first column of a table) to slide out a detail panel showing: definition, release frequency, whether the measure is YoY/MoM/level, the latest value, source status, the full source URL, and the last-verified date. Press Esc or click outside the panel to close it.

Navigation

  • Category tabs: Click any of the 8 category buttons to jump to that section
  • Scrolling: Tables scroll horizontally for months beyond the viewport
  • Source verification: Every cell value is a clickable link to the original data source, color-coded by verification status — see Section 4

4. How to Interpret the Data

Reading the KPI Cards

Each KPI card shows the most recent available value, not necessarily the current month. If June CPI hasn't been released yet, the card shows May CPI. Always check the small month label below the value to know which period you're looking at.

Source Status Color Coding

Every value in the dashboard is a link, and its color tells you how trustworthy the source is. A legend is shown just above the KPI cards. The four states are:

AppearanceMeaningWhat to do
Dark black, bold Confirmed source. Someone has checked that the linked URL shows this exact value for this exact month. Trust it. Safe to cite directly in reports or client conversations.
Light red, normal weight Broken or weak link. A source URL exists, but it points to a generic page (e.g. a portal homepage) rather than the specific release, or it hasn't been verified yet. Click through and check. If it's correct, mark it confirmed (see Section 9). If it's wrong, find the right page and update it.
Orange, normal weight No source. A value exists but no source URL was ever attached. Find the original release and add the source URL via Admin Mode.
Gray, italic, shown as "—" No value. This indicator has not been released yet for this month, or the data point was intentionally cleared. Nothing to do until the data is released. Not an error.
Tip: Hover over any value to see a tooltip confirming its status, or click the indicator name to open the detail drawer, which shows the full source status, source URL, and last-verified date for the latest month.

Data Tiers

Each data point in data.json has a tier field indicating source reliability:

Understanding Each Category

Monetary & Liquidity

  • Repo Rate: RBI's benchmark lending rate. Changes signal policy tightening (rate hike) or easing (rate cut). Updated on RBI MPC meeting dates (6 times/year).
  • System Liquidity: Net liquidity surplus/deficit in the banking system (₹ Lakh Crore). Positive = surplus (easy money), negative = deficit (tight money).
  • Credit Growth YoY: Year-on-year growth in bank lending. 12-15% is considered healthy; above 18% may signal overheating.
  • Deposit Growth YoY: Year-on-year growth in bank deposits. If credit growth significantly exceeds deposit growth, banks face funding pressure.
  • Credit-Deposit (CD) Ratio: Ratio of loans to deposits. Above 78% signals banks are stretched; below 72% suggests room for lending.

Institutional Flows

  • FII Net Flow: Foreign Institutional Investor buying/selling in Indian equities (₹ Cr). Positive = net buying (bullish signal), negative = net selling.
  • DII Net Flow: Domestic Institutional Investor flows. Often counter-cyclical to FII — DIIs buy when FIIs sell, providing market stability.
  • SIP Contribution: Monthly Systematic Investment Plan inflows into mutual funds. A measure of retail investor confidence and savings discipline.
  • Lumpsum Inflow: One-time mutual fund investments. Tends to spike when markets are perceived as cheap.
  • Redemption (Equity+Hybrid): Mutual fund outflows. Rising redemptions combined with falling SIPs can signal retail panic.

Production & Activity

  • PMI (Manufacturing/Services): Purchasing Managers' Index. Above 50 = expansion, below 50 = contraction. Above 55 = strong expansion.
  • Vehicle Sales: A proxy for consumer demand and credit availability. PV (passenger vehicle) sales reflect urban consumption.
  • Electricity Consumption: A real-time economic activity proxy — correlated with industrial output and GDP.
  • Steel Production YoY: Signals infrastructure and construction activity.
  • Fuel Consumption (Petrol/Diesel): Petrol reflects urban mobility/consumer activity; diesel reflects industrial, freight, and agricultural activity.

Inflation & Prices

  • CPI Inflation: The headline consumer inflation number. RBI's target is 4% (with ±2% tolerance band of 2-6%).
  • Core Inflation: CPI excluding food and fuel — shows underlying price pressure. More relevant for monetary policy than headline CPI.
  • WPI Inflation: Wholesale price index — a leading indicator for CPI. Divergence between WPI and CPI signals margin pressure on producers.
  • Food Inflation: Most volatile component of CPI. Driven by monsoon, supply disruptions, and global commodity prices.

Growth & Employment

  • GDP Growth YoY: Quarterly figure (not monthly) — only updates 4 times a year with a 2-month lag. The most comprehensive measure of economic health.
  • Unemployment Rate: CMIE monthly estimate. Below 7% is considered healthy for India. Watch the trend, not single-month readings.

Fiscal

  • GST Collection: Monthly gross GST revenue (₹ Cr). Reflects consumption activity and tax compliance. Crossing ₹2 lakh crore/month is considered strong.
  • GST YoY: Year-on-year growth in GST. Adjusts for the base effect — more meaningful than absolute numbers.
  • CapEx Outlay: Government capital expenditure. Higher CapEx = infrastructure investment and multiplier effects on GDP.
  • Current Account Deficit (% GDP): Trade balance indicator. Below -2.5% GDP is comfortable; beyond -3% raises funding concerns.

Money Market & Yields

  • 91-Day T-Bill: Short-term risk-free rate. Reflects near-term RBI policy expectations and liquidity conditions.
  • 364-Day T-Bill: Medium-term rate. Gap between 91D and 364D signals yield curve shape.
  • 10Y G-Sec Yield: The benchmark long-term rate. Rising yield = tightening financial conditions or inflation fears. Falling yield = rate cut expectations.

Currency & Forex

  • USD/INR: Exchange rate. Rising = rupee depreciation. Watch absolute level and rate of change.
  • GBP/INR: Relevant for UK-linked trade and NRI flows.
  • Forex Reserves: India's foreign currency reserves (USD Bn). Above $600Bn is considered comfortable. Declining reserves may signal RBI intervention to defend the rupee.

5. How to Analyze & Draw Inferences

Framework: Three-Layer Analysis

Use the dashboard in three passes, each building on the previous:

Pass 1 — Monthly Health Check (2 minutes)

Look at the 6 KPI cards for the latest readings. Ask: "Is anything flashing a warning?"

SignalWhat to WatchInterpretation
CPI > 6%Above RBI upper bandRate hike risk; debt funds vulnerable; equity multiples compress
CPI < 4%Below RBI targetRoom for rate cuts; positive for duration trades and equity valuations
PMI < 50Contraction territoryEarnings downgrades likely; defensive sectors outperform
FII Net Flow deeply negativeSustained selling (>₹10K Cr/month)Rupee pressure, large-cap drag, but DII counter-buying limits damage
USD/INR sharp move (>2% in a month)Rapid depreciationImport-heavy sectors hurt; IT/pharma benefit; RBI may intervene
GST YoY < 5%Slowing consumptionConsumer discretionary and FMCG earnings at risk

Pass 2 — Trend Spotting (5 minutes)

Use the trend chart dropdown to plot 3-6 month trajectories. Look for:

  • Inflection points: Is an indicator reversing direction? E.g., CPI turning up after months of decline = potential hawkish RBI pivot.
  • Divergences: Manufacturing PMI rising but vehicle sales falling = inventory build-up, not genuine demand. Credit growth rising but deposit growth stagnant = funding stress ahead.
  • Breakouts from range: 10Y G-Sec yield moving above its 6-month range = market pricing in tightening. GST collections consistently above ₹1.8L Cr = structural improvement in tax base.

Pass 3 — Cross-Category Narratives (10 minutes)

The highest-value analysis comes from connecting dots across categories. Here are key cross-reads:

Combine TheseTo AnswerExample Narrative
Repo Rate + CPI + 10Y Yield Where is monetary policy headed? If CPI is falling toward 4%, repo rate is at 6%+, and 10Y yield is declining → market expects rate cuts. Position for duration.
FII Flow + USD/INR + Forex Reserves Is the external sector stable? FII selling + rupee weakening + reserves declining → triple stress. If reserves are still above $600Bn, RBI has room to manage.
Credit Growth + CD Ratio + System Liquidity Can the credit cycle sustain? Credit growth at 16%, CD ratio at 80%, liquidity deficit → banks may slow lending. Watch for NBFC stress.
SIP + Lumpsum + Redemption What is retail sentiment? SIPs steady + lumpsum rising + redemptions falling = retail is confident. Good for mid/small-cap flows.
GST + Vehicle Sales + Electricity + Fuel How strong is domestic demand? All four rising together = broad-based demand recovery. GST rising but vehicle sales flat = services-led growth, not goods.
GDP + Unemployment + CapEx Is growth translating to jobs? GDP at 7%+ but unemployment rising = jobless growth, often driven by capital-intensive sectors. CapEx rising supports future job creation.
Analyst tip: The most actionable insight usually comes from a disagreement between related indicators. When everything points the same direction, markets have already priced it in. When two related indicators diverge, something is about to break one way — that's where the alpha is.

Seasonality to Keep in Mind

  • April GST: Typically elevated (year-end advance tax settlements). Don't mistake it for a trend.
  • Monsoon months (Jun–Sep): Food inflation can spike; fuel (diesel) demand for agriculture rises.
  • October–December: Festive season boosts vehicle sales, consumption, and GST. Compare YoY, not month-on-month.
  • March: Government CapEx spikes as ministries rush to exhaust budget allocations.
  • GDP: Q1 (Apr-Jun) often has base-effect distortions. Q3 (Oct-Dec) is the festive quarter.

6. Limitations & Caveats

Data Limitations

LimitationImpact
No real-time data Values are entered manually (except FX rates). The dashboard shows data as of the last update, not live feeds. Most macro indicators are released with a 2-4 week lag after month-end.
GDP is quarterly, not monthly GDP Growth only updates 4 times a year with a ~2 month lag. Empty cells for non-release months are expected, not errors.
CAD is quarterly Current Account Deficit data is released quarterly by RBI, similar to GDP.
Preliminary vs. revised numbers Some indicators (GDP, IIP, trade deficit) get revised weeks or months after the initial release. The dashboard stores the value at the time of entry — it does not auto-update with revisions.
No intra-month granularity Each indicator has one value per month. For daily/weekly series (FII flows, forex), the dashboard shows the monthly aggregate or end-of-month snapshot.
Source variability Different sources may report slightly different values for the same indicator (e.g., SIAM vs. media for vehicle sales). The data tier field helps assess reliability.

Technical Limitations

LimitationImpact
Manual update process Data freshness depends on someone updating the dashboard each month. There is no automated pipeline from source to dashboard (except FX rates via the Refresh button).
Single-user editing If two people edit simultaneously, whoever uploads last wins. There is no merge or conflict resolution.
No historical versioning in the UI Overwritten values are gone. GitHub's commit history serves as the audit trail, but the dashboard itself doesn't show past revisions.
Browser-only Charts don't render in email or PDF exports. For presentations, use screenshots or the Export to Excel feature.
Admin password is a UI gate, not true security The password hash is visible in the HTML source code. Real security comes from GitHub push access — only people with commit access can publish changes.
FX auto-fetch only The Refresh button can only auto-fetch USD/INR and GBP/INR (via open.er-api.com). All 32 other indicators must be entered manually due to CORS restrictions or paywall issues on data sources.

Analytical Limitations

  • No forecasting or modeling: The dashboard shows actuals only. It does not project, forecast, or compute derived metrics (e.g., real interest rates, output gaps).
  • No adjustments: Values are nominal (not seasonally adjusted, not inflation-adjusted). Be careful comparing months with known seasonal patterns.
  • Correlation ≠ causation: Cross-category analysis can reveal relationships, but the dashboard provides no statistical testing. Treat observed patterns as hypotheses to investigate, not conclusions.
  • 21+ months of history: Not enough for cycle-level analysis (interest rate cycles, credit cycles). Use external databases for multi-year trends.

7. One-Time Setup: GitHub Pages

Follow these steps once to get your dashboard live on the internet.

1 Create a GitHub Repository

  1. Go to github.com → click the + button (top right) → New repository
  2. Name it something like india-economic-pulse
  3. Set visibility to Public (required for free GitHub Pages)
  4. Check "Add a README file"
  5. Click Create repository

2 Upload Dashboard Files

  1. In your new repo, click Add file → Upload files
  2. Drag and drop these two files from the project folder (they live at the project root, not in a subfolder):
    • index.html
    • data.json
  3. Write a commit message like "Initial dashboard upload"
  4. Click Commit changes
Important: Both files must be at the root of the repository (not inside a subfolder). Confirm they appear as index.html and data.json directly in the repo file listing, not nested under a folder.

3 Enable GitHub Pages

  1. Go to your repo → Settings (gear icon)
  2. In the left sidebar, click Pages
  3. Under "Build and deployment":
    • Source: Deploy from a branch
    • Branch: main
    • Folder: / (root)
  4. Click Save
  5. Wait 1–2 minutes. Your dashboard will be live at:
    https://<your-username>.github.io/india-economic-pulse/

4 Verify

Open the URL in your browser. You should see the full dashboard with all data loaded. If you see a loading error, check that both files are in the root of the repository.

8. Monthly Update Workflow

Repeat these steps each month when new data becomes available (typically 1st–10th of each month).

1 Open Dashboard & Enter Admin Mode

  1. Open your live dashboard URL in a browser
  2. Click the lock icon (🔒) in the top-right corner
  3. Enter the admin password: Fincart@2026 (default)
  4. A green toolbar appears at the bottom — you're now in Admin Mode

2 Add the New Month (if needed)

Two options:

  • Option A — Refresh button: Click ⟳ Refresh data in the top nav. This auto-adds the current month, fetches FX rates, and enters Admin Mode. See Section 10 for details.
  • Option B — Manual: Click + Add Month in the admin toolbar → enter YYYY-MM format (e.g., 2026-08).

3 Edit Values

See Section 9 for detailed editing instructions.

4 Download Updated JSON

  1. Click ⬇ Download JSON in the admin toolbar
  2. A file named data.json downloads to your computer
  3. This file contains ALL data (not just your changes)
Warning: Edits exist only in your browser's memory. If you refresh or close the page before downloading, all edits are lost. Always download first.

5 Upload to GitHub

  1. Go to your GitHub repo
  2. Click Add file → Upload files
  3. Drag and drop the downloaded data.json
  4. It will show as "modified" (replacing the old file)
  5. Write a commit message, e.g.: Update data: Aug 2026 CPI, GST, PMI
  6. Click Commit changes

6 Verify

  1. Wait ~60 seconds for GitHub Pages to redeploy
  2. Open your dashboard URL and hard-refresh (Ctrl + Shift + R)
  3. Confirm the new values appear in the tables and KPI cards

9. Editing Values & Admin Mode

Entering Admin Mode

Click the 🔒 lock icon in the top-right nav bar and enter the password (Fincart@2026 by default). When admin mode is active:

  • A green toolbar appears at the bottom with edit count, Download JSON, and Add Month buttons
  • Table cells become clickable for editing
  • An "Exit admin mode" button appears in the top-right

Editing a Cell

  1. Navigate to the category tab containing the indicator
  2. Click any table cell — an edit dialog opens
  3. Enter the new value (numeric only — the dashboard handles formatting)
  4. Paste the source URL where you found the data (optional but strongly recommended for audit trail)
  5. Click Save (or press Enter)
  6. The cell updates immediately and gets a yellow highlight
  7. The change counter in the admin toolbar increments

Tips for Editing

  • Enter raw numbers — no commas, no currency symbols, no percent signs. E.g., enter 197000 not 1,97,000; enter 3.94 not 3.94%.
  • Leave the value field empty to clear a data point (the cell will show "—").
  • You can edit any month, not just the latest. Useful for correcting past values when revised data is released.
  • The chart re-renders automatically after each save.
Remember: All edits are in-memory only. You must click ⬇ Download JSON and upload to GitHub to make changes permanent.

How Newly Entered Values Are Colored

When you type a new value into a cell through Admin Mode, the dashboard does not automatically mark it "Confirmed" (black) — even if you paste a source URL. New entries always appear as:

  • Light red if you entered a source URL (status: not yet verified)
  • Orange if you left the source URL blank

This is intentional — it keeps "someone typed this in" visually distinct from "someone confirmed the link shows the exact figure," so nothing gets silently trusted.

Marking a Source as Confirmed (Turning it Black)

To promote an entry from red/orange to confirmed black, you edit the underlying data.json directly (this is a JSON field, not something exposed in the Admin Mode UI):

  1. Download the current data.json (or open the one you're about to upload) in a text editor
  2. Find the indicator and month, e.g.: 
    "cpi_inflation": {
  "2026-07": {
    "cpi_yoy_pct": 3.9,
    "source": "https://pib.gov.in/PressReleasePage.aspx?PRID=XXXXXXX",
    "source_status": "fixed",
    "last_verified": "2026-08-05"
  }
}
  3. Open the source URL yourself and confirm it shows this exact value for this exact month
  4. Add or update two fields on that month's entry:
    • "source_status": "fixed" — turns the cell black and bold
    • "last_verified": "YYYY-MM-DD" — today's date, shown in the detail drawer
  5. Save the file and upload it via Add file → Upload files on GitHub (Section 8, Step 5)
Note: If you never set source_status, the cell defaults to red (has a link) or orange (no link) — it will never silently become black. This is a manual, deliberate action by design.

10. Refresh Button: Auto-Fetch + New Month

The ⟳ Refresh data button (top-left, next to the title) performs the following in one click:

  1. Adds the current month to all indicators (if not already present). Safe to click multiple times — it won't add duplicate months.
  2. Auto-fetches USD/INR and GBP/INR exchange rates from the open.er-api.com API and fills them in.
  3. Lists all other indicators as "pending" in a result modal, so you know exactly what needs manual entry.
  4. Auto-enters Admin Mode so you can immediately start clicking cells to fill values.
Note: Only FX rates are auto-fetched because open.er-api.com is the only CORS-friendly, free API available. All other indicators (CPI, GST, PMI, etc.) come from government sources that don't provide CORS-enabled APIs, so they must be entered manually.

Typical monthly workflow: Click Refresh → review the fetched FX rates → click cells for CPI, PMI, GST, etc. → Download JSON → Upload to GitHub.

11. Changing the Admin Password

The default admin password is Fincart@2026. To change it:

  1. Choose your new password
  2. Compute its SHA-256 hash:
    • Online: Go to emn178.github.io/online-tools/sha256.html, type your password, copy the hash
    • Python: python -c "import hashlib; print(hashlib.sha256('YourNewPassword'.encode()).hexdigest())"
    • PowerShell:
    $bytes = [System.Text.Encoding]::UTF8.GetBytes("YourNewPassword")
$hash = [System.Security.Cryptography.SHA256]::Create().ComputeHash($bytes)
($hash | ForEach-Object { $_.ToString("x2") }) -join ""
  3. Open index.html in a text editor
  4. Find this line near the top of the <script> block: 
    const ADMIN_HASH = "ea04253a...c67b";
  5. Replace the hash string with your new hash
  6. Upload the updated index.html to GitHub
Security note: The admin password is a UI convenience gate, not true security. The real protection is GitHub push access — only repo collaborators can publish changes. Even if someone guesses the password, they can only edit in their own browser session.

15. Future Scope

The dashboard is designed to be extended incrementally. Below are potential enhancements, ordered by feasibility:

Near-Term (Low Effort)

EnhancementWhat It Would Do
More auto-fetch sources Add CORS proxy or serverless function to auto-fetch PMI (from S&P Global RSS), CPI (MOSPI API), and GST (PIB scraper) during Refresh. Reduces manual entry to ~15 indicators.
Month-on-month change indicators Show MoM % change alongside absolute values in the tables. Helps spot acceleration/deceleration without mental math.
Color-coded cells Green/red/amber heatmap in tables based on thresholds (e.g., CPI > 6% = red, < 4% = green). Instant visual signal.
Dark mode CSS toggle for dark theme. Useful for analysts working late or presenting on dark backgrounds.

Medium-Term (Moderate Effort)

EnhancementWhat It Would Do
Multi-indicator overlay charts Plot 2-3 indicators on the same chart (dual Y-axis) for correlation analysis. E.g., CPI vs Repo Rate, FII vs USD/INR.
Automated data pipeline GitHub Actions cron job that runs a Python script on the 1st and 15th of each month, fetching data from APIs and committing updated data.json automatically.
Commentary / notes per month Free-text notes field for each month — "RBI held rates despite CPI uptick" — providing narrative context alongside raw data.
PDF/PowerPoint export One-click export of KPI cards + selected charts as a formatted report for investment committee presentations.
Additional indicators IIP (Industrial Production), trade deficit, crude oil prices, Nifty 50 returns, India VIX — extending coverage to ~50 indicators.

Long-Term (Significant Effort)

EnhancementWhat It Would Do
Database backend Replace data.json with a database (Supabase / Firebase). Enables multi-user editing, revision history, role-based access, and real-time sync.
Alert system Email or Slack notifications when key indicators breach thresholds (e.g., CPI > 6%, FII selling > ₹20K Cr). Requires a backend.
Macro scoring model Composite macro health score (0–100) aggregating all indicators with weighted signals. Useful for systematic asset allocation.
Sector impact mapping Link macro indicators to sector performance — "which sectors benefit when CPI falls and credit growth rises?" — creating an investable output layer.
